          This "commit" method is the most significant operation in the TAOD project. Each MObject is identified by its FileType + Metadata combination. Jobber ensures that MObject under the "object" is unique by FileType + Metadata. If you try to commit a MObject with duplicating FileType + Metadata, the commit will be rejected.
